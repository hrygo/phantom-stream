## 3. 技术对抗 (Technical Confrontation)
*   **观测到的战术**:
    *   红方不再尝试生成合法的空压缩流，而是直接用 `0x00` 填充目标流内容。
    *   这种暴力填充导致所有目标流（Object 72, 76, 59, 60）的 zlib 头被破坏。
*   **采取的措施**:
    *   使用 `defender verify` 工具验证。
*   **验证结果**:
    *   **Anchor 1 (Attachment)**: 失败 (`zlib: invalid header`)。
    *   **Anchor 2 (SMask)**: 失败。
        *   工具尝试解压 SMask 流时，均报错 `zlib: invalid header`。
        *   由于流内容被全零填充，无法恢复任何原始数据。
*   **技术原理**:
    *   红方的“零字节填充”策略虽然简单粗暴，但在保持文件结构（xref 偏移量）不变的前提下，彻底破坏了流数据的完整性。
    *   由于 SMask 本身是辅助显示层，破坏它会导致部分图像显示异常（如透明度丢失），但文件依然能打开，符合红方的“结构卫生”原则。
